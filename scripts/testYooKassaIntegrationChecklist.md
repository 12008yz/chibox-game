# Чек-лист для тестирования интеграции YooKassa

## 1. Проверка webhook

- [ ] Убедиться, что webhook URL доступен и принимает POST запросы.
- [ ] Проверить, что подпись webhook проверяется корректно (X-Request-Signature-SHA256).
- [ ] Отправить тестовый webhook с помощью `generateYooKassaWebhookTestRequest.js`.
- [ ] Убедиться, что сервер отвечает статусом 200 и обновляет статус платежа в базе.
- [ ] Проверить логи на отсутствие ошибок при обработке webhook.

## 2. Тестовые платежи через API YooKassa

- [ ] Запустить скрипт `testYooKassaAuth.js` для создания тестового платежа.
- [ ] Убедиться, что платеж создаётся успешно (статус 200, корректный ответ).
- [ ] Проверить, что webhook получает уведомление о платеже и обновляет данные.
- [ ] Проверить в базе, что платежи имеют правильные статусы и связаны с пользователями.

## 3. Проверка данных в базе

- [ ] Проверить, что поля платежей заполнены корректно (payment_id, status, amount, user_id и т.д.).
- [ ] Убедиться, что статусы платежей обновляются после webhook.
- [ ] Проверить, что подписки пользователей активируются корректно.

## 4. Подготовка к работе с настоящей кассой

- [ ] Заменить тестовые ключи на реальные секретные ключи YooKassa.
- [ ] Проверить, что webhook и API работают с реальными ключами.
- [ ] Провести тестовые платежи в реальной среде.
- [ ] Убедиться в корректной обработке платежей и подписок.

## 5. Автоматизация тестирования (опционально)

- [ ] Написать интеграционные тесты для webhook и API платежей.
- [ ] Настроить CI/CD для автоматического запуска тестов.

---

Этот чек-лист поможет систематизировать тестирование и убедиться в корректной работе интеграции YooKassa.
